## 2FA (Двухфакторная аутентификация)

Дополнительный пароль для входа в аккаунт. Даже если кто-то получит SMS-код, без 2FA-пароля войти не сможет. Обязательно для купленных аккаунтов.

## API Hash / App ID

Пара идентификаторов приложения для работы с Telegram API. App ID — число, API Hash — строка. Влияют на траст сессии. Можно получить на my.telegram.org или использовать из официальных клиентов.

## Device ID

Уникальный идентификатор устройства. Telegram присваивает его на основе характеристик: модель, ОС, версия клиента. Если с одного Device ID входят в много аккаунтов — все они связываются между собой.

## DC (Data Center)

Дата-центры Telegram. Аккаунты привязаны к DC в зависимости от региона номера.

| DC | Регион |
|----|--------|
| DC1, DC3 | США |
| DC2, DC4 | Европа |
| DC5 | Азия |

## FLOOD_WAIT

Ошибка «подожди X секунд». Появляется при слишком частых запросах. Число после FLOOD_WAIT — сколько ждать. Повторные попытки увеличивают время.

## MTProto

Протокол Telegram для передачи данных. MTProto-прокси — специальные прокси, которые работают только с Telegram и маскируют трафик.

## Premium

Платная подписка Telegram. Аккаунты с Premium получают больше доверия от системы — спамер не будет платить за подписку.

## Trust Score

Внутренний рейтинг доверия аккаунта. Не показывается, но влияет на лимиты, частоту верификаций, скорость получения спамблоков. Повышается возрастом, стабильностью, отсутствием жалоб.

## Верификация

Запрос кода для подтверждения владения аккаунтом. Приходит при входе с нового устройства, смене региона, подозрительной активности.

## Окружение

Всё, что Telegram видит при подключении: IP, устройство, регион, timezone. Несоответствие параметров (например, немецкий IP + московское время) вызывает подозрения.

## Отлёжка

Период 3-5 дней после входа в аккаунт, когда нужно минимизировать активность. Telegram особенно внимательно следит за новыми сессиями.

## Прокси

Промежуточный сервер между вами и Telegram. Бывают SOCKS5 и MTProto. Telegram видит IP прокси, а не ваш.

## Резидентный IP

IP от домашнего провайдера. Самый надёжный тип — им пользуются обычные люди. Датацентровые и VPN-IP имеют худшую репутацию.

## Сессия

Авторизованное подключение к аккаунту. Создаётся при входе, сохраняется до завершения. Один аккаунт может иметь несколько сессий (телефон, компьютер, веб).

## Спамблок

Ограничение от Telegram за подозрительную активность. Бывает частичный (нельзя писать незнакомым) и полный (заблокирован). Проверить статус — @SpamBot.

## Тихий бан

Ограничение, о котором Telegram не сообщает. Сообщения отправляются, но не доходят до получателя. Проверить — @SpamBot.
